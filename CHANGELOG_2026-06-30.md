## 구현 내역

### 0. 주식 실시간 연동 (티커 → 현재가)
- **Yahoo Finance**(무료, 키 불필요) 시세 서비스 `app/services/prices.py` 신설.
  - `query1.finance.yahoo.com` chart 엔드포인트로 미국(AAPL)·한국(005930.KS)·코인(BTC-USD) 조회.
  - FX와 동일하게 **메모리+파일 캐시 + 오프라인 폴백** — 페이지 로드는 네트워크를 막지 않고, 캐시 가격으로 즉시 표시.
- 자산 목록 상단에 **🔄 가격 새로고침** 버튼 (`POST /assets/refresh-prices`) — 모든 주식 보유항목 시세 갱신.
- 주식 보유항목 저장 시 즉시 1회 시세를 받아 평가금에 반영.

### 1. 한 계좌에 주식 + 달러/원화 함께 입력
- **계좌(Asset) → 보유항목(Holding) 모델**로 전환.
  - `Holding`: `kind`(cash/stock), `currency`(KRW/USD), 현금=`amount`/`label`, 주식=`ticker`/`quantity`/`cached_price`.
  - 계좌 평가금 = 보유항목 합(주식=수량×현재가, 통화 환산 포함).
- 자산 추가/수정 폼에 **동적 보유항목 에디터**(`assets-form.js`): `＋현금` / `＋주식` 행 추가·삭제, 종류별 입력 토글.
- 데모: **토스증권 계좌** = USD 예수금 $1,200 + AAPL ×10(USD) + 삼성전자 ×5(KRW) 한 계좌.

### 2. 부동산 제외 순자산 (대시보드 & 리포트)
- `Category.is_real_estate` 플래그 추가(카테고리 편집기에서 토글, 부채는 불가).
- 대시보드에 **‘부동산 제외 순자산’ 카드** 추가(총자산·총부채·순자산·부동산제외 4카드).
- 스냅샷에 `real_estate_krw`·`net_worth_excl_re_krw` 컬럼 + 리포트 순자산 추이에 **부동산 제외 라인** 동시 표시.

### 3. 총자산 항목별 비율(%)
- 카테고리별 구성에 **점유율 %** 표시(대시보드 막대 옆 + 자산 탭 카테고리 헤더 칩).

### 4. 금액 밑 "몇억 몇만" 축약 표시
- `won_short` Jinja 필터 추가 (예: `129,925,000 → 1억 2,992만`).
- 대시보드 지표·카테고리 금액·자산 목록 평가금 아래 작은 보조 라벨로 표시.

### 5. 현금성 투자자산 대비 (중위험+고위험) 투자 비중
- 리포트에 **‘현금성 투자자산 대비 투자(중·고위험)’ 도넛** 추가.
  - 일반 ‘현금’은 제외하고 *현금성 투자자산* 카테고리만 분모에 포함(요청대로 “리얼 투자예수금 vs 투자금”).

### 6a. 목표 ↔ 자산 연동 (자동 합산)
- `Goal.linked_category_ids` / `linked_asset_ids` 추가.
- 목표 폼에서 **연동 카테고리/개별 자산을 체크**하면 현재 금액이 그 자산들의 실시간 합으로 자동 계산(미연동 시 수동 금액 폴백).
- 데모: ‘내 집 마련’=부동산+노후·안전자산 연동(196,800,000 / 65%), ‘비상금’=현금+현금성투자 연동(100%), ‘유럽여행’=수동.

### 6b. 리포트 비율 — 선택·조합형 비율 바 (Sector Allocation 스타일)
- 카테고리를 **체크해서 넣고 빼는** 가로 스택 비율 바 + 범례(각 항목 %·금액).
- 기본 선택 = **현금성 투자자산·중위험·고위험**(현금 제외) → 순수 투자 비중 즉시 확인, 통화 토글에도 반영.
